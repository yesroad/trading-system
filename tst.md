⚠️ 실거래 전 필수 단계 (추가 2-3개월 소요)

1. 백테스팅 (2-4주)

❌ 현재 계획에 없음
✅ 필요: 최소 2년 역사적 데이터로 전략 검증

- backtest-framework 스킬 활용
- Walk-forward 분석
- 슬리피지/수수료 반영

2. Paper Trading (1-2개월)

❌ 현재 계획에 없음
✅ 필요: 실시간 데이터로 모의 거래

- 실제 시장 조건에서 테스트
- 다양한 시장 국면 경험 (상승/하락/횡보)
- 예상 승률 vs 실제 승률 비교

3. 소액 실거래 (1개월)

✅ 필요: 계좌의 5-10%만 사용

- 심리적 적응 기간
- 실제 슬리피지 측정
- 긴급 중지 프로세스 검증

4. 누락된 중요 요소

A. 전략 검증

- 현재: AI 신뢰도 + 기술적 지표 블렌딩
- 필요: 역사적 성과 증명 (Sharpe ratio, Max Drawdown, Win rate)

B. 시장 조건별 대응

- 급등락 시 거래 중단 로직
- 유동성 부족 종목 필터링
- 뉴스 이벤트 시 거래 중지

C. 인간 개입 프로세스
// 예시: 긴급 중지 버튼
await updateSystemGuard({
trading_enabled: false,
reason: 'Manual halt by operator'
});

D. 모니터링 & 알림

- 실시간 Telegram 알림 (현재 있음)
- 대시보드 필요 (현재 계획에 없음)
- 일일/주간 성과 리포트

E. 법적/규정 준수

- 자동매매 신고 (한국: 금융위원회)
- 세금 처리 (양도소득세, 거래세)
- 증권사 약관 확인 (자동매매 허용 여부)

🎯 권장 로드맵

Phase 1-6: 기술 구현 (현재 계획) - 3개월

- 이 계획서대로 진행
- 목표: 기술적으로 완성된 시스템

Phase 7: 백테스팅 - 2-4주

1. 2년치 역사 데이터 수집
2. backtest-framework 스킬로 전략 검증
3. 최소 목표:
   - Sharpe Ratio > 1.5
   - Max Drawdown < 15%
   - Win Rate > 50%

Phase 8: Paper Trading - 1-2개월

1. 실시간 데이터로 모의 거래
2. 서킷브레이커 실제 발동 확인
3. 다양한 시장 조건 경험

Phase 9: 소액 실거래 - 1개월

1. 계좌의 5% 투입
2. 1개월 안정성 확인
3. 점진적 증액 (10% → 20% → 50%)

Phase 10: 본격 운영

✅ 3개월 이상 안정적 성과
✅ 예상 손실 범위 내 운영
✅ 긴급 상황 대응 검증 완료

📋 체크리스트 (실거래 전 필수)

- 백테스팅 완료 (Sharpe > 1.5)
- Paper Trading 2개월 (다양한 시장 조건)
- 긴급 중지 프로세스 검증
- 실시간 모니터링 대시보드 구축
- 법적 요구사항 확인 (자동매매 신고 등)
- 세금 처리 방안 마련
- 심리적 준비 (손실 감당 가능 금액 설정)
- 소액 실거래 1개월 (계좌의 5%)
- 전략 성과 검증 (Win rate, Sharpe, Drawdown)
