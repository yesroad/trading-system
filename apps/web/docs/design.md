# 트레이딩 모니터 대시보드 제작 요청서

## 1. 목적

첨부한 대시보드 이미지(UI 시안)를 기준으로  
**실시간 트레이딩 모니터 대시보드**를 구현한다.

## 디자인 참고

- /apps/web/docs/list.png 형태의 대시보드와 레이아웃을 구상한다
- 하단 카드 리스트는 10개 단위로 페이지네이션 처리한다
- /apps/web/docs/list.png는 카드 리스트의 우측 화살표를 누르면 펼쳐지는 모습이다

본 대시보드는:

- 코인 / 미장 / 국장 시장 상태를 한 눈에 보여주고
- AI 분석 결과 및 매매 위험도를 시각화하며
- 매매 결과(성과)를 기간별로 확인할 수 있어야 한다.

---

## 2. 참고 자료

- 첨부 이미지: **트레이딩 모니터 대시보드 UI 시안**
- 본 요청서는 **UI 구조 + 표시 데이터 정의** 중심이며,
  실제 데이터는 API 연동을 전제로 한다.

---

## 3. 전체 화면 구조

### 3-1. 상단 공통 영역 (시장 요약 카드)

CRYPTO / US / KR 3개 시장을 한 줄 카드 형태로 표시

#### 표시 항목

- 시장명 (CRYPTO / US / KR)
- 상태: 정상 / 지연 / 중단
- 최근 수집 시각 (N분 전)
- 최근 AI 분석 시각 (N분 전)
- 위험도 요약:
  - HIGH / MEDIUM / LOW 종목 개수

---

### 3-2. 시장 선택 탭

- 탭 순서: `[crypto] [kia] [yf]`
- 탭별 페이지 경로:
  - `/dashboard/crypto` -> CRYPTO 시장
  - `/dashboard/kia` -> KR 시장
  - `/dashboard/yf` -> US 시장
- 상단 공통 영역은 유지하고, 탭 본문만 페이지 단위로 전환

---

## 4. 시장 탭 상세 구성

### 4-1. 탭 상단 요약 줄

- 분석 대상 종목 수
- 최근 수집 시각
- 최근 AI 분석 시각

---

### 4-2. 위험도 분포 카드

3개의 카드로 표시:

- HIGH
- MEDIUM
- LOW

각 카드에는 **해당 위험도 종목 개수** 표시

---

### 4-3. 필터 영역

다음 필터를 제공:

- 전체
- HIGH
- MEDIUM
- LOW
- 보유 종목만

---

### 4-4. 종목 리스트 (핵심 영역)

#### 기본 표시 정보

- 종목 코드 (symbol)
- 보유 여부 배지
- 위험도 (HIGH / MEDIUM / LOW)
- 신뢰도 (confidence %)
- AI 요약 문장

#### 펼침 영역 (상세)

- AI 판단 근거 (2~3줄)
- AI 판단 결과 (ALLOW / CAUTION / BLOCK)
- 분석 기준 시각

※ raw 데이터(JSON)는 UI에 직접 노출하지 않음

---

## 5. 매매 결과(성과) 표시 영역

> 이미지에는 없지만 **필수 기능으로 추가 요청**

### 기간 선택

- 일간
- 주간
- 월간
- 전체

### 표시 지표

- 총 매매 횟수
- 수익 거래 수 / 손실 거래 수
- 누적 손익 (금액)
- 수익률 (%)

---

## 6. 데이터 신뢰 원칙

- UI는 서버에서 전달된 데이터를 그대로 표시
- 클라이언트에서 임의 계산/추정 금지
- 데이터 누락 시 `N/A` 표기
- 시간은 서버 UTC 기준, UI에서 로컬 변환

---

## 7. 반응형 요구사항

- 데스크톱 / 태블릿 / 모바일 대응
- 모바일에서는:
  - 카드 세로 스택
  - 종목 리스트 아코디언 방식 유지
